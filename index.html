    What is Functional IoT?

    The Internet of Things (or IoT for short) refers to uniquely identifiable objects and their virtual representations in an Internet-like structure. Imagine IoT devices are:

    • connected to the internet
    • developed in a short time
    • storing personal data
    • secure
    • more intelligence
    • inexpensive

    C language can design the IoT devices such like that? No, can't. We need the language that is good for system programming with strong type, to develop IoT device more safety more drastic approach.

    The Functional IoT is a wrestling mat that strong typed languages fight on, to become the champion of system programming language.

    What hardware platform will be supported?

    ARM Cortex-M

    • Spec: 32-bit, RAM-size 1KiB~256KiB
    • Debugger: ST-LINK/V2 (detail), CMSIS-DAP (detail)
    • Boards: mbed LPC1768, STM32F4DISCOVERY

    TI MSP430

    • Spec: 16-bit, RAM-size 128B~66KiB
    • Debugger: eZ-FET
    • Boards: MSP-EXP430FR5969

    Atmel AVR

    • Spec: 8-bit, Harvard architecture, RAM-size 64B~16KiB
    • Debugger: AVR Dragon (detail)
    • Boards: Arduino Mega 2560

    How to use?

    • ATS programing on Arduino (AVR)
    • ATS programing on ChibiOS/RT
    • ATS programing on mbed (ARM Cortex-M)
